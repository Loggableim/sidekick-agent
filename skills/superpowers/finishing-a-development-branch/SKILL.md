# Finishing a Development Branch

Finish with evidence, then offer the appropriate integration path.

1. Verify the requested tests, relevant regressions, and working-tree scope.
2. Summarize commits and any known baseline failures separately from new work.
3. Offer only applicable next actions: merge locally, open a review request, retain the branch, or clean an isolated worktree.
4. Do not delete branches, reset changes, or publish remotely without explicit authorization.

Use this after `verification-before-completion`, not as a substitute for it.
