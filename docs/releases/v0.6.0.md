# Sidekick v0.6.0 — Windows Install UX Release

**Tag:** `v0.6.0`
**Datum:** 2026-06-07
**Basiert auf:** `v0.5.0` (`129322f`)

---

## Ziel des Releases

Dieses Release bringt einen vollständigen **Windows One-Line PowerShell Installer** für Sidekick, inklusive Desktop Shortcut, WebUI Auto-Open und grundlegendem Installer-Hardening. Das Ziel war, den Einstieg auf Windows so einfach wie möglich zu gestalten — kein Admin, keine manuellen Schritte, nur ein Befehl in PowerShell.

## Änderungen im Detail

### `install.ps1` (1484 Zeilen)

Der Installer wurde adaptiert vom ausgereiften Sidekick-Vorgänger und für Sidekick angepasst. Er deckt den gesamten Provisionierungsprozess ab:

| Bereich | Beschreibung |
|---------|-------------|
| **Basis-Pfad** | User-lokale Installation unter `%LOCALAPPDATA%\sidekick\` — kein Administrator erforderlich |
| **Python-Provisionierung** | `uv`-basiert (Python 3.11+ via `uv python install`) |
| **Git-Provisionierung** | Automatischer PortableGit-Download bei fehlendem Git-System (ARM64/AMD64 aus GitHub Releases) |
| **Repository-Setup** | `git clone` oder `git fetch + reset` bei bereits vorhandenem Clone |
| **Virtuelle Umgebung** | Python `.venv` unter `%LOCALAPPDATA%\sidekick\sidekick-agent\.venv` |
| **Abhängigkeiten** | `pip install -e ".[all]"` — gestaffelt in 4 Tiers |
| **Desktop Shortcut** | `Sidekick.lnk` auf `%USERPROFILE%\Desktop\` → `sidekick dashboard` |
| **WebUI Auto-Open** | `Start-Process http://127.0.0.1:8787` nach Start von `sidekick dashboard` |
| **Update/Repair** | `-UpdateOnly`-Flag überspringt Prerequisite-Installationen |
| **Konfiguration** | `.env`, `config.yaml`, `SOUL.md` als Vorlagen (nur bei Erstinstallation) |
| **Gateway-Auto-Start** | Automatischer Gateway-Start, wenn API-Tokens konfiguriert sind |
| **Node.js** | Optional — kein automatischer Install; browser Tools bleiben deaktiviert |

## Bekannte Einschränkungen

- Kein `winget` / `scoop` / `chocolatey`-Package
- Kein GUI-Installer (ausschließlich PowerShell)
- Kein Windows CI in der GitHub-Actions-Pipeline (nur Linux + macOS)
- Node.js nicht auto-installiert (browser Tools optional)
- Gateway-Warnings (2 non-blocking) bei fehlenden Konfigurationen
