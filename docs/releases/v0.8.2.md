# v0.8.2 — Windows installer portable mode finalization

**Release date:** 2026-06-11

## Summary

Finalizes the Windows installer Portable mode implementation. The v0.8.1 release notes declared Portable mode complete, but the actual code was missing critical guards. This release closes all gaps between documented behavior and actual behavior.

## Changes

### Windows installer — Portable mode finalization

- **`-Mode Portable` now actually skips UAC.** The elevation check was changed from `if (-not $script:IsElevated)` to `if (-not $script:IsElevated -and $script:Mode -eq "Admin")`. Portable mode users no longer see a UAC prompt.
- **UAC denial offers Portable fallback.** When `-Mode Admin` elevation is denied, the error message now explicitly suggests running with `-Mode Portable -NoPrompt` instead of just saying "re-run from elevated PowerShell".
- **Install-SystemPackages respects Portable mode.** When `$script:Mode -eq "Portable"`, the function checks for existing ripgrep/ffmpeg but skips winget/choco/scoop install attempts (which typically require admin). Manual install instructions are shown for missing tools.
- **Comment block updated.** "Portable = future" → "Portable | Admin" reflecting that Portable mode is fully implemented.
- **Documentation comments** now explain both Admin and Portable modes, what each provides, and the limitations of Portable mode.

### Sidekick-Launcher.ps1 — uv pip install binding

- **Explicit `--python` flag.** The `uv pip install` call in `Ensure-PythonRuntime` now uses `--python "$venvPython"` to explicitly target the `.venv` Python, matching the pattern used in `install.ps1`.

### Documentation

- **README.md:** Added Portable mode bullet point, `%LOCALAPPDATA%\sidekick\` installation location, and note that Portable mode requires downloading the script first.
- **docs/troubleshooting.md:** New "Portable mode (no admin)" section with full instructions, limitations, and example commands.
- **docs/releases/v0.8.2.md:** This file.
