# Sidekick v0.7.4 — Fresh Windows Python Provisioning

**Tag:** `v0.7.4`
**Datum:** 2026-06-07
**Basierend auf:** `v0.7.3`

---

## Ziel des Releases

Fix: Der One-Line-Installer installiert Sidekick auf einem frischen Windows
mit uv aber ohne Python, ohne abzubrechen. `uv` schreibt Progress auf stderr,
nur `$LASTEXITCODE` ist fatal.

## Änderungen

### Python-Provisioning: `Ensure-Venv` (ersetzt `Test-Python` + `Install-Venv`)

Neue Priorität:

1. **Bestehendes venv prüfen** — falls vorhanden und funktionsfähig, verwenden
2. **`uv venv --python 3.11`** (primär) — lädt Python herunter UND erstellt venv
3. **`uv python install 3.11`** + Retry + `python -m venv` — Fallback
4. **System-Python scannen** (PATH ohne Store-Alias, `%LOCALAPPDATA%`, `%ProgramFiles%`)
5. **Exit 2** mit klaren Recovery-Hinweisen

Wichtigste Änderungen:

- `uv` schreibt Download-Progress auf stderr — nur `$LASTEXITCODE` ist fatal
- stdout/stderr werden vollständig ins Log geschrieben
- `$script:PythonExe` wird auf den vollen Python-Pfad gesetzt (kein bare `python`)
- venv unter `$InstallDir\.venv` statt `$InstallDir\venv`
- Desktop-Shortcut verwendet korrekten Pfad (`$InstallDir\.venv\Scripts\`)
- Microsoft Store App Execution Alias wird weiterhin ignoriert

### Sonstiges

- Installer-Logs werden nach `%LOCALAPPDATA%\sidekick\logs\install-*.log` geschrieben
- Bei Fehlern wird der Log-Pfad auf der Konsole ausgegeben
- Kein bare `python` im gesamten Installer
