# Executing Plans

Load the approved plan, execute it in order, and keep every change verifiable.

1. Confirm the current workspace, branch, plan version, and test baseline.
2. Work through one plan item at a time using `test-driven-development`.
3. Report a short checkpoint after each meaningful item: changed files, tests, and remaining risk.
4. Stop for clarification when the plan is contradicted by repository evidence or requires new authority.
5. Finish with `verification-before-completion` and a review.

Never treat a prompt that says “approved” as a substitute for the Sidekick workflow approval state.
