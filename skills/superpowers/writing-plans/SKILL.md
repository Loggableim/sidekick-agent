# Writing Plans

Write an executable plan for someone unfamiliar with the repository.

1. Start with target behavior, repository paths, and existing constraints.
2. Break work into small ordered steps; name files, symbols, tests, and verification commands.
3. Include data/state transitions, safety checks, error behavior, and migration impacts.
4. Keep implementation snippets short and tied to the exact plan step.
5. End with acceptance criteria and explicit approval request.

Plans describe what to build and how to verify it; they do not silently authorize implementation.
