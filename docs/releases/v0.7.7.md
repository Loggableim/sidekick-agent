# Sidekick v0.7.7 — PowerShell Parse Error Fix

**Tag:** `v0.7.7`
**Datum:** 2026-06-08

---

## Ziel des Releases

Der öffentliche One-Line-Installer war nach dem venv-Path-Fix (v0.7.6)
syntaktisch ungültig — PowerShell brach sofort beim `iex`-Parse ab.

## Problem

Die automatische Ersetzung in v0.7.6 erzeugte doppelte PowerShell-Quotes:

```powershell
$shortcut.TargetPath = ""$script:SidekickExe""     # FEHLER
$sidekickExe = ""$script:SidekickExe""              # FEHLER
```

PowerShell interpretiert dies als leeren String `""` gefolgt von `$script:SidekickExe""`
— ein unerwartetes Token.

## Fix

**2 Zeilen** in `install.ps1`:

| Zeile | Vorher | Nachher |
|-------|--------|---------|
| 1571 | `$shortcut.TargetPath = ""$script:SidekickExe""` | `$shortcut.TargetPath = $script:SidekickExe` |
| 1587 | `$sidekickExe = ""$script:SidekickExe""` | `$sidekickExe = $script:SidekickExe` |
