## Änderungen

### 1. WebUI-Fehlerbehandlung (Commit `...`)

**Problem:** API-Endpunkte gaben Fehler in 3 verschiedenen Formaten zurück
(`{"error": "..."}`, `{"ok": false, "error": "..."}`, `{"success": false, "error": "..."}`)
und viele leckten rohe `str(exc)`-Tracebacks.

**Lösung:**
- Neuer standardisierter Error-Helper `error_response()` in `web/api/helpers.py`:
  `{"ok": false, "error": {"code": "...", "message": "...", "details": ...}}`
- `_sanitize_error()` entfernt Filesystem-Pfade und sensible Pfade aus Fehlermeldungen
- `bad()` delegiert jetzt an `error_response()` — Legacy-Kompatibilität
- **24 `str(exc)`-Stellen** in `routes.py` auf den neuen Helper umgestellt
- Pre-existing Bug in `_run_lifecycle_health()` gefixt (`runs`-Liste nicht initialisiert)

### 2. Graceful Failures ohne API-Key

Alle Einstiegspunkte starten ohne API-Key ohne Traceback:

| Command | Vorher | Nachher | 
|---------|--------|---------|
| `sidekick --help` | ✅ OK | ✅ OK |
| `sidekick --version` | ✅ OK | ✅ OK |
| `sidekick doctor` | ⚠️ Exit 0 trotz Warnings | ✅ Exit 1 bei Warnings, Exit 2 bei Critical |
| `sidekick dashboard` | ✅ Startet | ✅ Startet |
| WebUI `/health` | ✅ OK | ✅ OK (Bug gefixt) |

### 3. WebUI HTTP Smoke-Test

**Neue Datei:** `tests/smoke_webui.py`

Startet den Server auf freiem Port und testet:

| Test | Ergebnis |
|------|----------|
| `/health` returns 200 | ✅ |
| Static asset (favicon) | ✅ |
| Session created via POST | ✅ |
| Sessions list | ✅ |
| Server stops cleanly | ✅ |

In `tests/smoke_all.py` integriert (17 Tests).

### 4. `sidekick doctor` mit Exit-Codes

**Datei:** `cli/doctor.py`

| Exit Code | Bedeutung |
|-----------|-----------|
| 0 | Alles gesund |
| 1 | Warnings (fehlende optionale Pakete, nicht-kritisch) |
| 2 | Critical (fehlende API-Keys, Config kaputt) |

Der Exit-Code-Support wurde durch Tracking von `_warning_count` und
`_fail_count` während der Doctor-Checks implementiert.

### 5. Troubleshooting-Dokumentation

**Neue Datei:** `docs/troubleshooting.md`

Themen:
- Installation / Fresh Clone
- Fehlende API-Keys / Provider-Credentials
- WebUI startet nicht (Port-Konflikte, fehlende Dependencies)

- Legacy `HERMES_*` Aliase
- Migration von `~/.hermes` nach `~/.sidekick` (Auto-Migration)
- Logs finden
- Smoke Tests ausführen
- FAQ

## Offene Risiken (v0.5.0 oder später)

- **Session-Layer:** `web.api.models.Session` (30+ Felder) und
  `shared.sessions.Session` (6 Felder) bleiben getrennt — das WebUI
  braucht agent state, streaming, compression anchor etc.
- **CI:** Nur Linux (ubuntu-latest). Windows/macOS fehlen.
- **Gateway:** Läuft stabil und ohne Warnings, aber Plattform-Tests fehlen.
