# Sidekick v0.7.0 — Packaging & Release UX

**Tag:** `v0.7.0`
**Date:** 2026-06-07
**Based on:** `v0.6.1`

## Release Goal

Improve the overall packaging and release experience: add a proper uninstall script, file-based install logging, formal release checklist, and comprehensive documentation for the install/update/repair/uninstall lifecycle.

## Changes

### `uninstall.ps1` — New uninstall script

A standalone PowerShell uninstaller that safely removes Sidekick from the system:

| Feature | Description |
|---------|-------------|
| `-RemoveUserData` switch | Default off — keeps `~/.sidekick` config. When set, removes it with a 5-second countdown warning |
| App directory removal | Removes `%LOCALAPPDATA%\sidekick\` entirely |
| Desktop shortcut | Removes `Sidekick.lnk` from the desktop |
| Environment variables | Cleans up `SIDEKICK_HOME` and `SIDEKICK_GIT_BASH_PATH` user env vars |
| PATH cleanup | Removes user PATH entries pointing to `%LOCALAPPDATA%\sidekick\git\` |
| Safety | Never touches ProgramFiles, system PATH, or anything outside `%LOCALAPPDATA%` |
| Colored output | Same Cyan/Green/Yellow/Red scheme as `install.ps1` |
| Exit codes | `0` = fully removed, `1` = partial (e.g. desktop shortcut not found) |

### Install Logging in `install.ps1`

All installer output is now written to both the console **and** a timestamped log file:

```powershell
%LOCALAPPDATA%\sidekick\logs\install-YYYYMMDD-HHmmss.log
```

- Log directory created automatically on first install
- All `Write-Info` / `Write-Success` / `Write-Warn` / `Write-Err` calls log to file
- On failure, the log file path is printed in the error message
- Zero change to existing console output behavior

### `docs/release-checklist.md` — New release process doc

A formal release checklist covering:

1. Smoke tests (`python tests/smoke_all.py` + `tests/smoke_webui.py`)
2. Branding audit (grep for legacy `hermes`/`nous`/`lastbrowser` references)
3. Version tagging (`git tag -a "v0.X.0"`)
4. Push tags (`git push origin master --tags`)
5. GitHub release creation
6. Installer one-line verification (test in VM)
7. Windows update verification (`git status` + re-run installer)
8. Config/state migration check (sessions survive update)
9. Readme/release notes update

## Known Limitations

- No `winget` / `scoop` / `chocolatey` package (still PowerShell-only)
- Uninstaller does not remove Node.js if Sidekick-managed (optional dep)
- `-RemoveUserData` requires explicit opt-in (5-second grace period)

## Upgrade Notes

### From v0.6.x

- Update by re-running the one-liner (idempotent):
  ```powershell
  irm https://raw.githubusercontent.com/Loggableim/sidekick-agent/master/install.ps1 | iex
  ```
- Or from an existing checkout:
  ```powershell
  cd %LOCALAPPDATA%\sidekick\sidekick-agent
  git pull
  ```
- Install logs are now written to `%LOCALAPPDATA%\sidekick\logs\install-*.log`

### Clean Uninstall

```powershell
# Remove app files, keep ~/.sidekick config
.\uninstall.ps1

# Remove everything including user data
.\uninstall.ps1 -RemoveUserData
```
