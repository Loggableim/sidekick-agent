# Requesting Code Review

Request review with a concise, evidence-backed change summary.

1. Review spec compliance: requirements, edge cases, and safety boundaries.
2. Review code quality: clarity, maintainability, tests, and regression risk.
3. Include changed files, verification output, scope limits, and known baseline failures.
4. Resolve or explicitly track every actionable finding before integration.

Do not ask reviewers to infer the intended behavior from a large diff alone.
