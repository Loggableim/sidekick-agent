# v0.8.4 — WebUI first-run onboarding fix

**Release date:** 2026-06-11

## Summary

Fixes the WebUI first-run onboarding wizard that was broken after the FastAPI server migration. The onboarding frontend called `/api/onboarding/*` endpoints that only existed in the old stdlib server (`web/server.py`), but the active dashboard uses the FastAPI server (`cli/web_server.py`) which had no onboarding routes — resulting in `{"detail":"Method Not Allowed"}` for every onboarding API call.

## Root cause

The onboarding frontend (`onboarding.js`) and backend (`web/api/onboarding.py`) were designed for the legacy stdlib HTTP server (`web/server.py` + `web/api/routes.py`). When `sidekick dashboard` migrated to FastAPI (`cli/web_server.py`), the onboarding routes were never ported, causing FastAPI's default 405 response.

## Changes

### FastAPI onboarding routes (cli/web_server.py)

- **Added 7 new route decorators** calling the existing `web.api.onboarding` handler functions:
  - `GET /api/onboarding/status` — Returns onboarding status, config paths, provider state
  - `GET /api/onboarding/oauth/poll` — Polls OAuth flow status
  - `POST /api/onboarding/oauth/start` — Starts OAuth device-code flow
  - `POST /api/onboarding/oauth/cancel` — Cancels OAuth flow
  - `POST /api/onboarding/setup` — Applies wizard-provided provider config
  - `POST /api/onboarding/complete` — Marks onboarding as completed
  - `POST /api/onboarding/probe` — Probes provider endpoint for model list

### Agent dir discovery (web/api/config.py)

- **Added Sidekick install paths** to `_find_agent_dir()`: `~/.sidekick/sidekick-agent`, `%LOCALAPPDATA%\sidekick\sidekick-agent` (Windows), `~/.local/share/sidekick-agent` (XDG). Previously only Hermes paths (`hermes-agent`) were searched.
- **Config path fallback**: Changed from `~/.hermes/config.yaml` to `~/.sidekick/config.yaml`.
- **Import check**: `verify_sidekick_imports()` now also imports `sidekick_app` alongside `run_agent`.

### Frontend fixes

- **`system.hermes_found` → `system.sidekick_found`** in `onboarding.js`: The backend returns `sidekick_found`, not `hermes_found`. This mismatch caused the "Nova — Missing or partially importable" warning on every first-run screen.
- **API error extraction**: `workspace.js` `api()` helper now also checks `j.detail` (FastAPI error format) before falling back to raw JSON text. Prevents `{"detail":"Method Not Allowed"}` from being displayed verbatim.

### No regressions

- Installer: install.ps1 unchanged (still parses, args parser, Admin/Portable mode, uv --python binding all preserved)
- No LastBrowser references added
- No runtime architecture changes
- No new features

## Files changed

- `cli/web_server.py` — 7 onboarding route decorators
- `web/api/config.py` — Agent dir discovery (Sidekick paths), config path fallback, import check
- `web/static/onboarding.js` — Field name fix: `sidekick_found` not `hermes_found`
- `web/static/workspace.js` — Error extraction: handle `detail` key
