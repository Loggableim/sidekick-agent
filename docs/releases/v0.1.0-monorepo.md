## Ziel des Releases

Erster veröffentlichter Stand des Sidekick-Monorepos. Ersetzt die historische
Trennung zwischen `cids-sidekick-agent` (Agent-Runtime + CLI + Tools) und
`cids-sidekick-webui` (WebUI-Server + Frontend) durch ein einziges installierbares
Paket mit drei First-Class-Surfaces: **CLI**, **TUI** und **WebUI**.

## Bekannte Einschränkungen

### Non-blocking Warnings

```text
Warning: config validation failed: cannot import name 'print_config_warnings'
         from 'runtime.config'
Warning: deprecation check failed: cannot import name 'warn_deprecated_cwd_env_vars'
         from 'runtime.config'
```

Diese 2 Meldungen erscheinen beim Start von `runtime.gateway.run` und sind
harmlos. Die Funktionen existierten im alten `sidekick_cli.config` und werden
beim nächsten Gateway-Refactor portiert oder entfernt.

### Gateway await-Bug (gefixt)

Der Bug `await outside async function` in `runtime/gateway/run.py` war ein
pre-existing Bug im Original-Source und wurde in diesem Release isoliert
behoben (`asyncio.run_coroutine_threadsafe` für Pool-Acquire/Release in
Sync-Callback).

### Session-Vertrag

Zwei Session-Modelle koexistieren:
- `shared/sessions.py` — kanonisches Modell mit `retry_last`, `undo_last`,
  `session_status` (für CLI/TUI)
- `web/api/session_ops.py` — WebUI-spezifische Wrapper mit Agent-Lock
  und Stream-Lifecycle (delegiert generische Ops an `shared.sessions`)

Beide teilen sich denselben Storage-Pfad (`~/.sidekick/state/webui/sessions/`).
Zusammenführung in `v0.2.0` geplant.

### CLI-Help-Strings

Die Subcommand-Beschreibungen in `sidekick --help` erwähnen noch teilweise
"Sidekick". Diese sind kosmetisch und werden in `v0.2.0` bereinigt.

### WebUI localStorage-Keys

Die WebUI verwendet noch `sidekick-theme`, `sidekick-skin` etc. als
localStorage-Keys. Funktionieren einwandfrei, aber Kosmetik für `v0.2.0`.
