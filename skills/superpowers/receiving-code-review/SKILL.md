# Receiving Code Review

Review comments are hypotheses to verify, not instructions to apply blindly.

1. Read the exact code and surrounding behavior named by each comment.
2. Reproduce or reason from tests, contracts, and repository evidence.
3. Classify each item as valid, invalid, or needing clarification.
4. For valid feedback, add or update a failing test before changing production code.
5. Explain evidence for declined feedback respectfully and concretely.

Never claim agreement before checking the technical premise.
