# Sidekick v0.7.4 — Windows Python Provisioning Fix

**Tag:** `v0.7.4`
**Datum:** 2026-06-08

---

## Ziel des Releases

Frischer Windows-Installer mit `uv` aber ohne Python soll nicht mehr mit
Exit 2 abbrechen. Der Installer nutzt `uv` um Python zu provisionieren,
ohne dass der Microsoft Store App Execution Alias oder fälschlich als
Fehler interpretierte Progress-Ausgaben den Installationsprozess stören.

## Fehlerursache

`$ErrorActionPreference = "Stop"` (Zeile 23) macht PowerShell aus **jedem**
stderr-Output externer Programme einen terminierenden Fehler. `uv python install`
schreibt Download-Progress ("Downloading cpython-3.11...") auf stderr, der
via `2>&1` ins PowerShell-Exception-System geriet — der `catch`-Block fing
den Progress als Fehler, bevor `$LASTEXITCODE` je geprüft wurde.

## Änderungen

### install.ps1: stderr via Temp-Datei isolieren

| Problem | Lösung |
|---------|--------|
| `$ErrorActionPreference = "Stop"` macht stderr zu Exception | `Invoke-External` Helper-Funktion |
| `2>&1` mischt Progress in Exception-Stream | stderr via `[System.IO.Path]::GetTempFileName()` umgeleitet |
| `$LASTEXITCODE` wurde nie erreicht | Exit-Code wird jetzt korrekt ausgewertet |

**Neue Helfer-Funktion `Invoke-External`:**
```powershell
function Invoke-External {
    param([string]$FilePath, [string[]]$ArgumentList)
    $stderrFile = [System.IO.Path]::GetTempFileName()
    try {
        $output = & $FilePath @ArgumentList 2>$stderrFile
        $ec = $LASTEXITCODE
        $errText = Get-Content $stderrFile -Raw -ErrorAction SilentlyContinue
        return @{ Output = $output; ExitCode = $ec; Stderr = $errText }
    } finally {
        Remove-Item $stderrFile -Force -ErrorAction SilentlyContinue
    }
}
```

**Betroffene Aufrufe:**
- `uv venv --python 3.11 <venv>` (Zeile 222)
- `uv python install 3.11` (Zeile 262)
- `uv python install 3.11` Retry (Zeile 294)

### Fallback-Reihenfolge (unverändert seit v0.7.3)

| Stufe | Pfad | Beschreibung |
|-------|------|-------------|
| 1 | Existierendes venv | `.venv\Scripts\python.exe` prüfen |
| 2 | **`uv venv --python 3.11`** | Python-Download + venv in einem Schritt |
| 3 | `uv python install 3.11` + venv | Mit einmaligem Retry |
| 4 | System-Python scannen | PATH ohne Store-Alias, `%LOCALAPPDATA%`, `%ProgramFiles%` |
