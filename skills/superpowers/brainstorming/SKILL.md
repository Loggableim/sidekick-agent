# Brainstorming

Turn a request into an approved design before implementation.

1. Inspect the relevant Sidekick workspace, conventions, and constraints.
2. Ask one focused question at a time only when an answer materially changes scope.
3. Present 2-3 viable approaches with trade-offs; recommend one.
4. Describe architecture, data flow, failure handling, and tests in small sections.
5. Ask for explicit design approval.

Do not write implementation code before approval. For very small, unambiguous fixes, state the assumption and proceed with a focused test.

## Handoff

After approval, use `writing-plans` for multi-step work and `test-driven-development` before implementation.
