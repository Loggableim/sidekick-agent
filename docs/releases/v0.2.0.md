# Sidekick v0.2.0 — Rebrand & Cleanup Release

**Tag:** `v0.2.0`
**Datum:** 2026-06-07
**Commit:** `5f9edfe`
**Vorgänger:** `v0.1.0-monorepo` (`ddd8208`)

---

## Ziel des Releases

Abschluss des Rebrandings von Hermes auf Sidekick auf allen user-facing
Oberflächen. Entfernung der letzten artefaktischen `cli_backup/`- und
Storage-Key-Altlasten.

---

## Änderungen

### 1. CLI-Help-Texte — Hermes → Sidekick (13 Stellen)

| Datei | Änderung |
|-------|----------|
| `cli/main.py` | `help=`-Strings: "Hermes" → "Sidekick" (6 Stellen) |
| `cli/main.py` | `description=`-Strings: "Hermes" → "Sidekick" (2 Stellen) |
| `cli/main.py` | Docstrings: "Hermes" → "Sidekick" (3 Stellen) |
| `cli/main.py` | Inline-Kommentare: "Hermes + Honcho" → "Sidekick + Honcho" (2 Stellen) |

**Ergebnis:** `sidekick --help` zeigt kein user-facing "Hermes" mehr.
Verbleibende "HERMES_*" im Help-Text sind dokumentierte Legacy-Env-Var-Referenzen
und bleiben als Kompatibilitätshilfe erhalten.

### 2. WebUI localStorage — Migration `hermes-*` → `sidekick-*`

**23 Keys** in **17 Dateien** migriert:

| Kategorie | Beispiel |
|-----------|----------|
| Theme/Skin | `hermes-theme` → `sidekick-theme` |
| UI-State | `hermes-webui-session` → `sidekick-webui-session` |
| Panel-Layout | `hermes-webui-workspace-panel` → `sidekick-webui-workspace-panel` |
| TTS-Einstellungen | `hermes-tts-voice` → `sidekick-tts-voice` |
| Sprach-Einstellung | `hermes-lang` → `sidekick-lang` |
| HTML-IDs | `hermesSessionMeta` → `sidekickSessionMeta` |
| CSS-Klassen | `hermes-action-grid` → `sidekick-action-grid` |
| Theme-Color Meta | `hermes-theme-color` → `sidekick-theme-color` |

**Migrations-Shim** (`web/static/boot.js`): Läuft einmalig beim ersten Laden nach
dem Update, kopiert alle existierenden `hermes-*`-Werte nach `sidekick-*` und
setzt Flag `sidekick-migrated-v1`. Bestehende User-Settings gehen nicht verloren.

### 3. `cli_backup/` entfernt

| Datei | Größe | Typ |
|-------|-------|-----|
| `cli_backup/__init__.py` | 1 KB | Forwarder |
| `cli_backup/main.py` | ~11.9 KB | Alter Argparser-Baseline |

**Begründung:** Der alte Baseline-Argparser war nur während der CLI-Migration
als Referenz notwendig. Seit `v0.1.0-monorepo` läuft der komplette CLI-Stack
über die produktive `cli/` (77 Module). Kein Rollback-Szenario mehr.
