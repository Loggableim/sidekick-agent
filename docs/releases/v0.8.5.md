# v0.8.5 ? Superseded WebUI transition

**Release date:** 2026-06-11

This note is retained for the release timeline only. Its transitional API
proxy design is no longer supported.

## Current architecture

Sidekick serves the WebUI and all API routes from one FastAPI application. The
route bridge invokes the established handlers in-process, including streaming,
cookies, headers, and session state. No second HTTP server, random backend port,
or inter-process proxy belongs to the release architecture.

## Verification

The current release contract is covered by the FastAPI bridge tests, the
dashboard smoke test, and the browser WebUI smoke test.
