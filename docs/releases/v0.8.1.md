# v0.8.1 — Windows installer consistency release

**Release date:** 2026-06-10

## Summary

Consistency and correctness fixes for the Windows PowerShell installer after the transition to admin/UAC mode. Also fixes port documentation, flag parsing, and uv binding.

## Changes

### Windows installer

- **Admin/UAC is now official and documented.** README and troubleshooting guide reflect that the standard installer requires admin rights. The `launcher.bat` comment block explains why (hosts alias, environment setup, optional toolchains, uninstall entry).
- **Real `$args` flag parser** replaces the old `param()`-incompatible approach. The script now supports:
  - `-UpdateOnly` — Update an existing install, skip prerequisites
  - `-NoVenv` — Skip venv creation
  - `-SkipSetup` — Skip the setup wizard
  - `-SkipOptionalTools` — Skip Node.js, ripgrep, ffmpeg checks
  - `-NoPrompt` — Skip all Read-Host prompts (auto-accepts)
  - `-NoDoctor` — Skip post-install doctor check (forward-looking)
  - `-Surface Browser|Standalone|CliOnly` — Control what gets installed/started
  - `-Mode Admin|Portable` — Future portable mode wiring
  - Unknown flags print usage and exit cleanly with exit code 1.
- **All `uv pip install` calls now explicitly target `.venv`** via `--python "$script:PythonExe"` instead of relying on `$env:VIRTUAL_ENV`.
- **Raw installer URL normalized:** Elevated-installer download uses `/$Branch/install.ps1` instead of `/HEAD/install.ps1`.
- **Optional tools are truly non-blocking.** `-SkipOptionalTools` skips Test-Node, Install-SystemPackages, and Install-NodeDeps. winget/choco/scoop failures never block core install.
- **`-Surface CliOnly`** skips the WebUI start entirely.
- **`-NoPrompt`** replaces all Read-Host calls with auto-"y" answers.

### Port consistency

- **All user-facing documentation now references port 9119** (matching the Windows installer/launcher). The CLI default port remains 8787; this is documented in README.
- Docs updated: README.md, docs/troubleshooting.md, docs/testing/windows-installer.md, and all release notes.
