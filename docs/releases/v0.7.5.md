# Sidekick v0.7.5 — Fix uv stderr Handling on Clean Windows

**Tag:** `v0.7.5`
**Datum:** 2026-06-08

---

## Ziel des Releases

`uv`-Download-Progress (stderr) soll nicht mehr fälschlich als Exception
behandelt werden, wenn `$ErrorActionPreference = "Stop"` gesetzt ist.
Stattdessen werden stdout und stderr sauber getrennt via `Start-Process`
mit `-RedirectStandardOutput`/`-RedirectStandardError` in Tempfiles.

## Änderungen

### install.ps1: Neue `Invoke-External`-Helper-Funktion

**Problem:**
Auf sauberem Windows mit `$ErrorActionPreference = "Stop"` führte das
bisherige `2>&1`-Merging dazu, dass `uv`'s Download-Progress auf stderr
als PowerShell-Exception gewertet wurde — `catch`-Blöcke feuerten, obwohl
`uv` mit Exit-Code 0 beendet war. Der bisherige Workaround (separates
Tempfile für stderr pro Aufruf) war fragil und inkonsistent.

**Lösung:**
Neue Funktion `Invoke-External` (nach den Write-*-Helfern, vor `Install-Uv`):
- Startet Prozesse via `Start-Process` mit getrennten Streams
  (`-RedirectStandardOutput`/`-RedirectStandardError` in Tempfiles)
- Liest stdout/stderr nach Prozessende ein
- Loggt alles automatisch ins Installationslog (`[CMD:stdout]`,
  `[CMD:stderr]`, `[CMD:exit]`)
- Gibt ein Hashtable mit `ExitCode`, `Stdout`, `Stderr` zurück
- Räumt Tempfiles im `finally`-Block auf
- Wirft niemals eine Exception bei stderr-Output — Exit-Code ist das
  einzige Erfolgskriterium

### install.ps1: `Ensure-Venv` auf `Invoke-External` umgestellt

- **Stufe 2** (`uv venv --python`): Statt `try/catch` + manuellem
  Tempfile-Management → `Invoke-External`, `.ExitCode`-Prüfung
- **Stufe 3** (`uv python install` + Retry): Gleiches Pattern —
  konsequent `Invoke-External` statt `& $UvCmd ... 2>&1`
- **Stufe 4** (System-Python-Scan): Unverändert (nutzt `& $exePath --version
  2>$null` für einfache Versionsabfragen — das ist sicher)
- **Keine `try/catch`-Blöcke** mehr um uv-Kommandos
- **Kein `$LASTEXITCODE`** mehr für uv — stattdessen `$result.ExitCode`
- Paths mit Spaces (`$VenvPath`) werden korrekt gequotet via
  `` "`"$VenvPath`"" `` im `-ArgumentList`-Array

### Bereinigung von Pre-Existing Bugs

- Der Retry-Block in Stufe 3 referenzierte fälschlich `$installExit`
  (den Exit-Code des ersten Versuchs) statt `$installExit2` — korrigiert
- Fehlende stderr-Logs im Retry-Block ergänzt

## Smoke/Validierung

- PowerShell Parse: Brace/Paren/Bracket balanciert
- `Invoke-External` definiert (vor `Install-Uv`, nach Write-*-Helfern)
- `Ensure-Venv` nutzt `Invoke-External` für alle uv-Aufrufe
- Kein `2>&1` mehr bei uv-Kommandos in `Ensure-Venv`
- Alle `Invoke-WebRequest`/`Invoke-RestMethod` mit `-UseBasicParsing`
- Keine user-facing Hermes/LastBrowser

## Bekannte Einschränkungen

- System-Python-Scan findet nur Python 3.10+ (nicht 3.9 oder älter)
- `uv venv` kann auf Systemen ohne Internet keine Python-Downloads durchführen
- `$script:UvCmd` muss gesetzt sein bevor `Ensure-Venv` aufgerufen wird
  (wird durch `Install-Uv` garantiert)
